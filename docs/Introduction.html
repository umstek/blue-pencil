blue-pencil: Scriptable Shredding System


CAUTION:

This is an alpha testing release of the software.


What is blue-pencil?

blue-pencil is a software which is designed to shred files.


What happens when we delete a file?

When we delete a file with the help of a file browser (i.e. Windows Explorer), the file is not actually deleted. But the mark (i.e. index) specifying where the file is and the size of the file and the filename etc. is deleted. So, we can use a normal recovery software (e.g.: undelete) to recover the file. If the file is not overwritten by another file, we can recover it.


So what can we do to make it unrecoverable?

What we do is overwriting the file with some values, so normal recovering software will not be able to recover it. However, a microscopic analysis on the disk can be done to recover the file. So, the file is still recoverable. Overwriting the file multiple times may reduce this risk of recovering. Several standards (or algorithms) are present to shred file. Most of them use pseudo-random numbers to overwrite the file.


Why use blue-pencil?

What is special with blue-pencil is the scripting support, which enables the user to create their own algorithm to shred files. And also we can edit present algorithms and customize them.

So we can call this software a scriptable shredding system. The blue-pencil script is basically based on XML (eXtensible Mark-up Language). It supports basic logical operations, mathematical operations (mathematical operations: not yet implemented) and conditions. There are no loops in this scripting language but loops are still accessible via functions.
